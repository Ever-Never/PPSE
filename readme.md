PPSE Applet包含个人化命令及选择指令（为了方便调试加了一条INS=0xB0的调试指令）  

**个人化指令**  
个人化前需要建立安全通道：INS=0x50、INS=0x82，由于打开安全通道的密钥由安全域管理，故这两条指令需要交给安全域来处理：processSecurity方法。  
建立安全通道时可能带有线路保护，故需要通过unwrap来解密数据。  
个人完成之后不能重复个人化，通过isEnd来标志个人化是否完成，同时此标志位可以保证StoreData指令掉电情况下程序的鲁棒性：StoreData执行后如果FCI中的A5长度为0则个人化过程中可能掉电了，
需要（先删除应用）重新个人化。  

**选择指令**  
组织FCI然后返回，其FCI包含84和A5两部分，84部分通过API getAID来组织，A5部分是通过个人化指令写入的，需要从个人化数据中组织。  

说明：此Applet的选择指令部分可以优化，PPSE的AID是固定的那么FCI的84部分就是固定的，个人化完成后A5部分也是固定的了，可以设置两个数组，
一个final的常量数组放84，一个数组放A5，组织数据的时候在RAM中组织即可。
